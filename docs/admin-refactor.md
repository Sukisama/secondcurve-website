# 管理后台重构说明

## 主要变更

### 1. 数据源迁移
- **之前**: 使用 localStorage 存储数据
- **现在**: 使用 Supabase 数据库

### 2. 权限控制
- 只有 `admin` 和 `super_admin` 角色可以访问管理后台
- 未登录用户会被重定向到登录页面
- 无权限用户会被重定向到 403 页面

### 3. 功能增强

#### 实战案例管理
- ✅ 创建新案例
- ✅ 编辑现有案例
- ✅ 删除案例
- ✅ 查看案例列表（按创建时间倒序）
- ✅ 支持字段：标题、分类、描述、作者、标签、概述等

#### 用户管理
- ✅ 查看所有用户
- ✅ 编辑用户信息（姓名、邮箱、角色、头像）
- ✅ 修改用户角色（guest/member/vip/admin/super_admin）
- ⚠️ 不能删除用户（保护数据完整性）
- ⚠️ 不能创建用户（用户需通过注册页面创建）

#### 活动管理
- ✅ 创建新活动
- ✅ 编辑现有活动
- ✅ 删除活动
- ✅ 查看活动列表（按活动日期排序）
- ✅ 支持字段：标题、描述、地点、日期、最大人数、价格、状态

#### 数据导出
- ✅ 导出所有数据为 JSON 文件
- ✅ 自动过滤敏感信息（用户 ID）
- ✅ 文件名包含导出日期

## 使用方法

### 1. 访问管理后台
```
URL: /admin
```

### 2. 权限要求
- 必须登录
- 用户角色必须是 `admin` 或 `super_admin`

### 3. 标签切换
- 实战案例：管理案例数据
- 用户管理：管理用户数据和角色
- 活动管理：管理线下活动

### 4. 创建/编辑数据
1. 在左侧表单填写信息
2. 标有 `*` 的字段为必填项
3. 点击"添加"或"保存"按钮提交

### 5. 删除数据
1. 点击右侧列表项的删除图标
2. 确认删除操作
3. 数据将从数据库中永久删除

### 6. 导出数据
1. 点击右上角"导出数据"按钮
2. 自动下载 JSON 文件

## 技术实现

### 依赖项
- `@supabase/supabase-js`: Supabase 客户端
- `lib/auth/permissions`: 权限验证
- `components/Toast`: 提示消息

### 数据库表
- `cases`: 案例数据
- `profiles`: 用户数据
- `events`: 活动数据

### RLS 策略
管理后台依赖 Supabase 的 Row Level Security 策略：
- `admin` 和 `super_admin` 可以管理所有数据
- 普通用户只能查看公开数据

## 注意事项

1. **用户不能删除**: 为了保护数据完整性，用户数据不能通过管理后台删除
2. **用户不能创建**: 新用户必须通过注册页面创建
3. **数据验证**: 所有操作都有数据验证，确保数据完整性
4. **错误处理**: 所有数据库操作都有错误处理和用户提示
5. **并发安全**: 提交按钮有防重复提交机制
