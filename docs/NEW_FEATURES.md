# 新功能部署说明

本文档说明如何部署和使用第二曲线网站的最新功能。

## 功能概览

### 1. 个人资料和名片编辑页面合并
- 统一的编辑页面，包含所有个人资料和名片信息
- 标签页式设计，分为基本信息、社交链接、展示设置三个部分
- 共用昵称和头像
- 在个人中心有"编辑个人资料和名片"快捷入口

### 2. 论坛点赞和收藏功能
- 在帖子详情页可以点赞和收藏帖子
- 点赞和收藏按钮带有计数显示
- 在个人中心可以查看所有点赞和收藏的内容
- 点赞和收藏数据实时更新

### 3. 个人中心贡献统计
- 自动统计用户发布的帖子数量
- 自动统计用户发布的案例数量
- 自动统计用户参与的活动数量
- 在个人中心正确显示这些数据

### 4. 首页数据动态化 + 活动报名功能
- 首页案例和活动数据从数据库动态读取
- 活动报名页面支持填写报名信息
- VIP会员可免费报名活动
- 报名信息自动记录到数据库

### 5. 成员编号系统
- 自动为每个注册用户生成唯一成员编号
- 编号格式：DEQX10001
- 在个人中心和名片页显示编号
- 确保编号唯一性

## 部署步骤

### 第一步：执行数据库迁移

1. 登录 Supabase Dashboard
2. 进入 SQL Editor
3. 执行 `/docs/migration-new-features.sql` 文件中的 SQL 脚本
4. 等待执行完成，确认没有错误

### 第二步：部署前端代码

```bash
# 确保所有依赖已安装
npm install

# 本地测试
npm run dev

# 构建生产版本
npm run build

# 部署到生产环境
# 根据你的部署平台进行操作
```

### 第三步：验证功能

1. **测试个人资料编辑**
   - 访问 `/profile/edit`
   - 检查所有标签页功能
   - 测试保存功能

2. **测试点赞和收藏**
   - 访问任意论坛帖子
   - 点击点赞和收藏按钮
   - 访问 `/profile/likes-favorites` 查看记录

3. **测试贡献统计**
   - 发布一个帖子或案例
   - 访问 `/profile` 查看统计数字

4. **测试活动报名**
   - 访问首页或 `/events`
   - 点击活动报名按钮
   - 填写报名信息并提交

5. **测试成员编号**
   - 注册新用户
   - 查看个人中心是否显示成员编号
   - 查看名片页是否显示成员编号

## 数据库表结构变更

### 新增表

1. **post_likes** - 帖子点赞表
   - `id`: UUID 主键
   - `post_id`: 帖子ID
   - `user_id`: 用户ID
   - `created_at`: 创建时间

2. **post_favorites** - 帖子收藏表
   - `id`: UUID 主键
   - `post_id`: 帖子ID
   - `user_id`: 用户ID
   - `created_at`: 创建时间

### 修改表

1. **profiles**
   - 新增 `member_code` 字段（成员编号）

2. **posts**
   - 新增 `likes_count` 字段（点赞数）
   - 新增 `favorites_count` 字段（收藏数）

## 注意事项

1. **数据迁移**
   - 执行SQL脚本前建议先备份数据库
   - 脚本会自动为现有用户生成成员编号

2. **权限设置**
   - 确保RLS策略正确设置
   - 只有会员及以上角色才能点赞和收藏

3. **性能优化**
   - 已添加必要的数据库索引
   - 使用触发器自动更新计数

4. **兼容性**
   - 保持了对旧数据的兼容
   - 新功能不影响现有功能

## 问题排查

### 点赞/收藏按钮无响应
- 检查用户是否已登录
- 检查用户角色是否为会员及以上
- 查看浏览器控制台是否有错误

### 成员编号未显示
- 检查数据库中 profiles 表是否有 member_code 字段
- 确认是否执行了迁移脚本
- 检查现有用户是否已生成编号

### 活动报名失败
- 检查是否已登录
- 确认活动是否还有名额
- 查看是否已经报名过该活动

## 技术栈

- **前端**: Next.js, React, Tailwind CSS
- **后端**: Supabase (PostgreSQL, Auth, RLS)
- **数据库**: PostgreSQL with Row Level Security
