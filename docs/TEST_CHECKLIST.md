# 功能测试清单

本文档列出了所有新增功能的测试步骤和预期结果。

## 准备工作

1. 确保已执行数据库迁移脚本 `/docs/migration-new-features.sql`
2. 启动开发服务器 `npm run dev`
3. 准备测试账号（至少需要：普通用户、VIP用户、管理员）

---

## 功能1：个人资料和名片编辑页面合并

### 测试步骤

1. 登录账号
2. 访问 `/profile`
3. 点击"编辑个人资料和名片"按钮
4. 检查页面是否显示三个标签页：基本信息、社交链接、展示设置

### 测试点

- [ ] 基本信息标签页
  - [ ] 可以上传/更换头像
  - [ ] 可以修改昵称（必填）
  - [ ] 可以修改个人简介
  - [ ] 可以填写公司、职位、城市
  - [ ] 可以填写技能标签和兴趣爱好

- [ ] 社交链接标签页
  - [ ] 可以填写微信、微博、GitHub、Twitter、LinkedIn、个人网站

- [ ] 展示设置标签页
  - [ ] 可以选择是否在成员墙展示
  - [ ] 可以选择是否在需求广场展示
  - [ ] 勾选后在需求广场展示后，可以填写需求信息

- [ ] 保存功能
  - [ ] 点击保存按钮后，数据保存成功
  - [ ] 保存后跳转回个人中心
  - [ ] 再次编辑时，数据正确回显

- [ ] 兼容性
  - [ ] 访问旧路径 `/card/edit` 自动重定向到 `/profile/edit`
  - [ ] 昵称和头像在个人中心和名片页保持一致

### 预期结果

- 所有字段可以正常编辑和保存
- 数据在不同页面间保持一致
- 标签页切换流畅，无卡顿

---

## 功能2：论坛点赞和收藏功能

### 测试步骤

1. 访问 `/forum`
2. 点击任意帖子进入详情页
3. 测试点赞和收藏功能

### 测试点

- [ ] 点赞功能
  - [ ] 未登录时点击提示登录
  - [ ] 登录后可以点赞
  - [ ] 点赞后按钮变红，数字+1
  - [ ] 再次点击取消点赞，按钮恢复，数字-1
  - [ ] 点赞数正确显示在帖子列表

- [ ] 收藏功能
  - [ ] 未登录时点击提示登录
  - [ ] 登录后可以收藏
  - [ ] 收藏后按钮变黄，数字+1
  - [ ] 再次点击取消收藏，按钮恢复，数字-1
  - [ ] 收藏数正确显示在帖子列表

- [ ] 个人中心查看
  - [ ] 访问 `/profile/likes-favorites`
  - [ ] 点赞标签页显示所有点赞过的帖子
  - [ ] 收藏标签页显示所有收藏过的帖子
  - [ ] 点击帖子可以跳转到详情页

### 预期结果

- 点赞和收藏按钮响应灵敏
- 数字实时更新
- 个人中心正确显示历史记录

  - [ ] 点击案例可以跳转到详情页

  - [ ] 活动卡片显示日期、标题、地点
  - [ ] 点击报名按钮跳转到报名页

- [ ] 活动报名页
  - [ ] 显示活动详细信息（时间、地点、费用）
  - [ ] VIP用户显示"VIP免费"标识
  - [ ] 表单自动填充用户信息
  - [ ] 可以填写姓名、手机号、公司、职位、备注
  - [ ] 提交报名成功
  - [ ] 检查报名记录是否保存在数据库

- [ ] 满员检查
  - [ ] 活动满员后，报名按钮显示"已满员"
  - [ ] 不能重复报名同一活动

### 预期结果

- 首页数据从数据库正确加载
- 报名流程顺畅
- VIP免费功能正常

## 功能5：成员编号系统

### 测试步骤

1. 注册一个新用户
2. 查看个人中心和名片页

### 测试点

- [ ] 自动生成编号
  - [ ] 新注册用户自动获得成员编号
  - [ ] 编号格式为 DEQX + 数字
  - [ ] 编号唯一，不重复

- [ ] 个人中心显示
  - [ ] 访问 `/profile`
  - [ ] 在用户信息区域显示成员编号
  - [ ] 格式：成员编号：DEQX10001

- [ ] 名片页显示
  - [ ] 访问任意用户的名片页 `/card/[id]`
  - [ ] 在姓名下方显示成员编号
  - [ ] 格式正确

- [ ] 现有用户
  - [ ] 执行迁移脚本后，现有用户都有编号
  - [ ] 编号连续递增

### 预期结果

- 所有用户都有唯一的成员编号
- 编号在个人中心和名片页正确显示
- 编号格式统一
